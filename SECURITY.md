# Security and Privacy

POS may contain sensitive personal data, including goals, health notes, life plans, calendar events, reminders, relationships, and reflections.

## Principles

- Private by default.
- User control over exports and deletion.
- Minimize sensitive data collection.
- Keep API keys out of the client app in production.
- Use a backend for AI and third-party integrations.
- Avoid exposing health or identity data in logs.

## Reporting Issues

For now, open a private security issue or contact the maintainer directly before public disclosure.
