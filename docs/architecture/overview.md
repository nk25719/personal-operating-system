# Architecture Overview

POS should be built as a platform, not only a mobile app.

## Clients

- iOS app first
- Future web app
- Future desktop widgets
- Future voice assistant interface

## Core Services

- Identity service
- Planning service
- Habit service
- Knowledge service
- Wellbeing service
- Environment and integrity service
- Automation service
- AI advisor service

## Product Layering

The visible app should remain simple: Today, Capture, Modules, Settings.

The underlying system can be complex, but complexity should be hidden behind automation and optional modules.
